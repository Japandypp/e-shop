----------------------------------------------------------------------
   			ACTUALIZAR SUDO / BASE DE DATOS
---------------------------------------------------------------------

1. Actualizar y mantener actualizado el software
	
	sudo apt update && sudo apt upgrade -y
	
2. Actualizar las librerias que se usaran en un proyecto, en este caso actualizamos python y sus herramientas basicas

	sudo apt install python3 python3-pip python3-venv git -y
	
3. Instalar MySQL o MariaDB

	sudo apt install mariadb-server mariadb-client -y

4. Ejecutar para activar la base de datos mariadb

	a. sudo systemctl start mariadb
	
	Comando para que siempre este activo mariadb cuando se inicia la pc
	
	b. sudo systemctl enable mariadb
	
5. Crear a la base de datos y un usuario para el proyecto 

	a. sudo mysql -u root -p
	
	b. CREATE DATABASE ecommerce_db; (crear la base de datos)

	c. CREATE USER 'ecommerce_user'@'localhost' IDENTIFIED BY '123456'; (crear un usuario, el 123456 es la clave)
	
	d. GRANT ALL PRIVILEGES ON ecoomerce_db.* TO 'ecommerce_user'@'localhost'; (comando para relacionar y dar permisos de que puede hacer todo en la base de datos con ese usuario)
	
	e. FLUSH PRIVILEGES;
	
	f. EXIT;
	
6. Crear la carpeta del proyecto 

	a. mkdir e-shop
	
	b. cd e-shop
	
7. Crear el entorno virtual de python, activar e instalar paquetes

	a. python -m venv venv
	
	b. source venv/bin/activate (activar mi entorno virtual)

	c. pip install flask flask-sqlalchemy flask-migrate pymysql python-dotenv flask-login

8. Creamos el repositorio

	git init
	
9. Este comando sirve para ignorar los paquetes que no usamos en linux y se puedan usar en otro sistema operativo

	a. echo "venv/" > .gitignore
	
	b. echo ".env" >> .gitignore
	
10. Congelar las librerias que tengo ahora y las guarda en un documento .txt

	pip freeze > requirements.txt

11. Enviamos el primer comentario al proyecto

	a. git add .
	
	b. git status
	
	c. git commit -m "Setup inicial del proyecto"

----------------------------------------------------------------------
   	PRACTICA 06 - Configuración de autenticación
----------------------------------------------------------------------

1. Dentro del proyecto y activado mi entorno virtual instalare una libreria que me ayudara con la 
	autenticación de la aplicacion
	
	pip install flask-login
	
2. Nos vamos a app/__init__.py inicializamos e incluimos la libreria flask login

	from flask_login import LoginManager
	
3. Agregar una instalancia en app/__init__.py
	
	login_manager = LoginManager()
	
	login_manager.init_app(app)
	
4. Configuramos el Login Manager

	login_manager.login_view = 'auth.login'
	
	login_manager.login_message = 'Inicia sesión para continuar'
	
	login_manager.login_message_category = 'warning'
	
5. Instalar la libreria 

	pip install flask-wtf email-validator
	
6. Agregamos las librerias al archivo requirements

	pip freeze > requirements.txt
